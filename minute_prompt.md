# 議事録作成アプリ 仕様書（Draft）

## 1. プロジェクト概要

Teamsの会議録画・録音ファイルから、AIを用いて「議題」「主な発言」「決定事項」を抽出し、ブラウザ上での編集を経てローカルフォルダに保存するWebアプリケーション。

## 2. システム構成・技術スタック

* **フロントエンド / バックエンド:** Streamlit (Python)
* **AI モデル:** * 音声認識：OpenAI Whisper API
* 要約・構造化：OpenAI GPT-4o


* **音声処理:** pydub (ffmpeg)
* **外部連携:** ローカルフォルダ保存
* **データ保存:** SQLite（メタデータ管理用）※保存期間3ヶ月

---

## 3. 機能要件

### 3.1 データ入力機能

* **ファイルアップロード:** Teamsから出力された音声（mp3/m4a）または動画（mp4）を受け付ける。
* **会議情報入力:** サイボウズOfficeの予定情報をテキストエリアにコピペ入力できる。

### 3.2 音声前処理機能

* **音声抽出:** 動画ファイル（mp4）から音声トラックのみを抽出。
* **自動分割:** OpenAI APIの制限（25MB）を超えるファイルに対し、無音区間考慮または一定時間ごとに分割処理を行う。

### 3.3 AI 議事録生成機能

* **文字起こし:** Whisper APIによる高精度なテキスト化。
* **構造化要約:** GPT-4oを用い、以下のフォーマットで出力。
1. **議題の説明:** 会議の目的や概要
2. **主な発言:** 重要なやり取りの要約
3. **決定事項:** 確定したタスクや合意点



### 3.4 編集・プレビュー機能

* **Webエディタ:** AIが生成した結果を、ローカルフォルダ保存前にブラウザ上で修正できる。

### 3.5 外部出力機能

* **ローカルフォルダ保存:** 編集後の内容を指定したローカルフォルダに保存し、保存先のパスをユーザーに提示する。

---

## 4. データ管理・セキュリティ

* **保存期間:** アップロードされたファイルおよび生成されたテキストデータは、作成から**3ヶ月**で自動削除するロジックを実装する。

---

## 5. 画面設計案 (Streamlit)

1. **サイドバー:** OpenAI APIキー設定、保存先フォルダの指定。
2. **メインエリア（上部）:** ファイルアップロード + サイボウズ情報のペーストエリア。
3. **メインエリア（中部）:** 「議事録生成」ボタン ＋ 処理プログレスバー。
4. **メインエリア（下部）:** 生成された議事録の編集用テキストエリア ＋ 「ローカルフォルダへ保存」ボタン。

---
